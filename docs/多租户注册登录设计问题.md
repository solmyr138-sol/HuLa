# 多租户注册登录设计问题

## 背景

修复"同一手机号在不同企业注册"功能时，发现登录流程存在设计缺陷。

## 已修复的问题

### 1. 代码层 — `checkMobile()` 全局查重
- `DefUserServiceImpl.checkMobile()` 查手机号时没有 `tenant_id` 条件，全表查重
- 注册接口 `/anyTenant/registerByEnterpriseMobile` 走免登录路径，无租户上下文
- `DefUserMapper` 有 `@InterceptorIgnore(tenantLine = "true")`，MyBatis-Plus 拦截器不会自动追加 `tenant_id`
- **修复**：新增 `checkMobileInTenant(mobile, tenantId, id)` 方法，`register()` 和 `registerImByMobile()` 改用该方法

### 2. 数据库层 — 唯一索引
- `def_user` 表唯一索引 `uk_user_mobile` 原为 `(system_type, mobile)`
- **修复**：改为 `(system_type, mobile, tenant_id)`

### 3. IM 子系统 — `existsByEmailAndIdNot()` 全局查重
- IM `User` 实体继承 `Entity`（非 `TenantEntity`），不受租户拦截器保护
- `UserDao.existsByEmailAndIdNot()` 查 email（实际存手机号）时无 tenant 过滤
- **修复**：新增 `existsByEmailAndTenant(uid, email, tenantId)` 方法，`UserServiceImpl.register()` 改用该方法

## 未解决的设计问题：登录时如何区分企业？

### 现状

`DefUserManagerImpl.findUserForLogin()` 查询条件只有 `system_type + mobile/username`，没有 `tenant_id`。如果同一手机号在多个企业注册，`getOne(..., false)` 随机返回其中一条记录，用户无法控制登录哪个企业。

```java
// DefUserManagerImpl.java line 116-131
public DefUser findUserForLogin(Integer loginType, String account, boolean emailLogin) {
    // ... 只按 systemType + account 查询，无 tenantId 条件
    user = getOne(Wrappers.<DefUser>lambdaQuery()
            .eq(DefUser::getSystemType, loginType)
            .eq(DefUser::getMobile, account), false);
    return user;
}
```

### 可选方案

1. **登录时也要求输入企业邀请码**（或企业编号）——用企业码 + 手机号 + 密码三要素登录，最简单直接
2. **手机号登录后弹出企业选择界面**——先按手机号查出关联的所有企业，让用户选
3. **登录后在首页/设置里提供「切换企业」功能**——默认登录上次使用的企业，可以手动切换

### 待决策

选定方案后需同步修改：
- 服务端：登录接口、`findUserForLogin` 查询逻辑
- 客户端：登录页面 UI（移动端 + 桌面端）
- 数据库：可能需要 `def_user_tenant_rel` 关系表（方案2/3）
